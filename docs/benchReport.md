# Bench Report — SensorTelemetry

## 1. Цель
Измерить производительность программы при анализе аномалий по скользящему окну и вычислении медианы на выборке JSON файлов.

## 2. Методика
- Использован встроенный Debug Mode программы
- Измерение времени медианы и скользящего окна для одного файла
- Измерение общего времени анализа для нескольких файлов
- Тестовые файлы: 60–600 ts/value на файл, окно 20 точек

## 3. Результаты

### Микрозамеры на одном файле
| Кол-во записей | Медиана (µs) | Скользящее окно (µs) | Время анализа (ms) |
|----------------|---------------|---------------------|------------------|
| 60             | 17            | 20                  | 0.04             |
| 600            | 164           | 159                 | 740              |

### Общие замеры для нескольких файлов
| Файлов проверено | Время анализа (ms) |
|-----------------|------------------|
| 100             | 6680             |
| 1000            | 54181            |
| 10000           | 514324           |

### Расчетное время для 100000 файлов
> Примерное время: ~5 143 240 ms ≈ 1 час 25 минут

---

## 4. Анализ
- Микрозамеры показывают, что **вычисление медианы и оконных метрик для одного файла крайне быстро**
- Основное время при обработке большого количества файлов занимает **чтение JSON и перебор данных**
- Влияние размера окна и медианы на маленьких файлах практически несущественно
- Зависимость времени анализа от количества файлов и количества записей **почти линейная**

## 5. Вывод
Программа демонстрирует **масштабируемость почти линейного типа**.  
Возможные оптимизации:
- Оптимизация чтения JSON (потоковый разбор, меньше копий данных)
